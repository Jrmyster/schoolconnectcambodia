# Chouy Sala (ជួយសាលា / Help School)

## Overview
Chouy Sala is a full-stack web application designed to connect rural Cambodian high schools with potential donors and NGOs. Its primary purpose is to facilitate funding and resource acquisition for schools in need, promoting transparency and community involvement. The platform aims to bridge the resource gap for Cambodian schools, enabling better educational opportunities and fostering sustainable development in rural areas.

## User Preferences
The user prefers clear and concise communication. They value detailed explanations for complex features and architectural decisions. The user is open to iterative development but wishes to be consulted before any major changes are implemented. They prefer that the AI prioritizes performance optimizations, especially for mobile and low-bandwidth environments, and ensures robust error handling and graceful fallbacks. The user wants the AI to strictly adhere to the defined architectural patterns and technology stack.

## System Architecture
The application is a pnpm monorepo using TypeScript, built on Node.js 24.

### Technical Implementations
- **Authentication**: Cookie-based sessions with `express-session`, `bcryptjs` for password hashing, and custom `AuthContext`. Includes a secure "Forgot Password" flow. In production, `SESSION_SECRET` is required (server fails fast if unset), `cookie.secure` is enabled, and `app.set("trust proxy", 1)` is applied so the secure flag works behind the platform's TLS-terminating proxy.
- **Admin Authorization**: Backend admin endpoints (e.g. `/api/admin/weekly-metrics`, `/api/admin/weekly-summary`) are gated by a dedicated `requireAdmin` middleware in `artifacts/api-server/src/middleware/rbac.ts` that returns 401 (no/stale session) or 403 (non-admin). Frontend `/admin*` routes are wrapped in `<AdminRoute />`.
- **Mobile Menu Accessibility**: Hamburger button exposes `aria-expanded` / `aria-controls` with bilingual labels, sized to a 44×44 touch target (`p-3` around a 20px icon); the mobile menu container has `role="dialog"` and an `aria-label`; pressing **Escape** or navigating to a new route auto-closes it.
- **Performance / 3G readiness**: Every page in `App.tsx` is loaded via `React.lazy()` so visitors only download the JavaScript for the page they open. A small bilingual `<PageLoading />` fallback wraps the `<Switch>`. Heavy 3D modules (`SolarSystem3D`, `HistoryGlobe`) are also dynamically imported inside their pages.
- **Viewport & Zoom**: `index.html` uses `<meta name="viewport" content="width=device-width, initial-scale=1.0">` — the previous `maximum-scale=1` was removed because it blocked pinch-to-zoom (an a11y violation).
- **Dark-mode Contrast**: Hero/secondary text on dark pages (`SpacePage`, `CinematographyPage`, `OceanographyPage`) uses `text-white/75–85` (and `text-cyan-100/80–85`) instead of the previous `text-white/35–55`, meeting WCAG AA contrast against the dark backgrounds.
- **SEO / Crawler hygiene**: `public/robots.txt` allows the public site but disallows `/admin`, `/admin/dashboard`, `/dashboard`, and `/school-inbox` from search-engine indexes (defence in depth — those routes are also gated server-side).
- **Global Atlas / World Directory** (`/study-center/global-atlas`): Offline-first bilingual reference of 50 countries (Asia 18, Europe 15, Africa 6, Americas 8, Oceania 3). Source data lives in `src/data/countriesData.json` (flag emoji, EN+KH name, capital, population, languages, government, GDP, literacy rate, religion, 2–3 wildlife, 1–2 landmarks, fun fact). Page (`src/pages/GlobalAtlas.tsx`) ships a search bar (matches EN, KH, capital, languages — diacritic-insensitive), 6 continent filter chips with live counts, a responsive card grid, and an accessible modal (`role="dialog"`, dual-id `aria-labelledby` for EN+KH, Escape/backdrop close, focus management, body scroll lock). A prominent gold "View Timeline · មើលតារាងពេលវេលា" CTA in the hero links to the World Timeline page (below). All UI labels (visible text AND aria-labels) are strictly bilingual EN+KH side-by-side. Linked from the Study Center navbar dropdown as "World Directory 🌍". To add a country, append a record to `countriesData.json`; the chip counts and search index update automatically.
- **World Timeline / History & Progress** (`/study-center/world-timeline`): Offline-first vertical timeline of 22 historical milestones from the Agricultural Revolution (~10,000 BCE) to mRNA vaccines (2020), including the 1113 CE building of Angkor Wat. Source data: `src/data/timelineData.json` (year, era, region, icon, EN+KH headline & description, optional `connectedTo` array for cause-and-effect links). Page (`src/pages/WorldTimeline.tsx`) renders a parchment/sepia layout with a vertical center line, alternating left/right cards, and a sticky filter bar (7 region chips × 8 era chips). Cards fade in on scroll via `IntersectionObserver`, but default to `visible=true` so the page still renders if JS fails — `useLayoutEffect` flips animation on (and visibility off) before the first paint on JS-capable clients to avoid a flash. "Connected To" buttons trigger smooth `scrollIntoView` plus a 1.8 s ring-pulse highlight on the target card. All visible labels and aria-labels are paired EN+KH (no `kh ? ... : ...` toggling); region/era labels live in `regionLabel*` / `eraLabel*` helpers so future enum values stay localized.
- **Global Search**: `GlobalSearch.tsx` is powered by `fuse.js` (fuzzy, typo-tolerant, Unicode-safe — works for both English and Khmer queries) over a merged index built from two sources: (1) the legacy hand-curated array in `src/data/searchIndex.ts` and (2) any auto-discovered `*.search.ts` files co-located next to a page or component. To add a new entry to the global search, create `src/pages/MyPage.search.ts` exporting `searchMeta: SearchEntry` (see header doc in `src/data/searchAutoIndex.ts`); discovery happens at build time via `import.meta.glob` so lazy-loaded pages stay lazy and HMR works automatically. Hash-fragment hrefs (e.g. `/study-center/philosophy#free-will-vs-determinism-title`) navigate to the path AND preserve the hash in the URL bar so destination components can react to it (see PhilosophyMap's hashchange-aware deep-link effect for the pattern).
- **Photo Upload**: Supports both camera capture and file uploads.
- **API Routes**: Standard RESTful endpoints for schools, needs, authentication, and password management.
- **Database**: PostgreSQL with Drizzle ORM.
- **Validation**: Zod for schema validation.
- **API Codegen**: Orval generates React Query hooks and Zod schemas from an OpenAPI spec.

### System Design Choices
- **Monorepo Structure**: Organized into `artifacts`, `lib`, and `scripts`.
- **TypeScript Configuration**: Uses composite projects with `tsconfig.base.json` and project references.
- **Build System**: `esbuild` for production bundles, `tsc` for type checking.
- **State Management**: `Zustand` for language toggling.
- **Error Handling**: `WebGLErrorBoundary` and `detectWebGL()` for 3D rendering.
